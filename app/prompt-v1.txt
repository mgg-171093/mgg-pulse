You are a senior software architect and UI/UX engineer.

Your task is to design and generate a complete Windows desktop application called **MGG Pulse**.

This application is a configurable, rule-based user activity simulator to prevent session timeouts in remote environments.

---

# 🧱 ARCHITECTURE

Use:

* Language: C#
* Framework: .NET 8
* UI: WinUI 3 (Windows App SDK)
* Pattern: MVVM
* Architecture: Hexagonal (Ports & Adapters) + SOLID principles

---

## Project Structure

Create a clean solution with:

* MGG.Pulse.Domain
* MGG.Pulse.Application
* MGG.Pulse.Infrastructure
* MGG.Pulse.UI

---

## Core Concepts

The app must be built as a **rule-based simulation engine**, not a simple loop.

---

# 🧠 DOMAIN LAYER

Define:

Entities:

* SimulationConfig
* SimulationState

Value Objects:

* IntervalRange (min/max)

Enums:

* SimulationMode (Intelligent, Aggressive, Manual)
* InputType (Mouse, Keyboard, Combined)

---

# 🔌 PORTS (INTERFACES)

* IInputSimulator
* IIdleDetector
* ILoggerService
* IConfigRepository

---

# ⚙️ APPLICATION LAYER

Implement use cases:

* StartSimulationUseCase
* StopSimulationUseCase
* EvaluateSimulationCycle

---

## Rule Engine (CRITICAL)

Create:

* IRule interface
* Implement rules:

  * IdleRule
  * AggressiveModeRule
  * IntervalRule

Create RuleEngine that evaluates all rules.

---

## Execution Engine

Implement a background worker using Task + CancellationToken:

Loop:

1. Wait for next interval (randomized if configured)
2. Evaluate rules
3. Execute simulation if allowed

---

# 🧩 INFRASTRUCTURE

Implement:

* Win32InputSimulator (SendInput)
* Win32IdleDetector (GetLastInputInfo)
* JsonConfigRepository
* FileLoggerService
